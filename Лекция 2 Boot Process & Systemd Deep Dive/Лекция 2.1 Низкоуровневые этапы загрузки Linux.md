#### Unified Kernel Image (UKI) в Ubuntu

- UKI **объединяет** ядро, initramfs и параметры cmdline **в единый .efi** файл;
- Ubuntu 24.04 LTS внедряет поддержку UKI для архитектуры amd64;
- Обеспечивает **строгое соблюдение подписей ядра** (kernel signature enforcement);
- Устраняет **необходимость в тяжеловесном** загрузчике GURB2;
- Позволяет загружаться **напрямую** через UEFI или легковесный systemd-boot;


#### Роль загрузчика (GRUB2 / systemd-boot)

Задача 1: **Инициализация** видеорежима и базового окружения;
Задача 2: Чтение конфигов (grub.cfg или записей loader);
Задача 3: Передача **параметров** ядру (например, root=UID...);
Задача 4: **Загрузка** бинарного файла **ядра** (vmlinuz) в RAM;
Задача 5: **Загрузка** образа **initramfs** в смежный блок памяти;


#### Операция Switch Root

- После обнаружения реального диска он **монтируется** в каталог /new_root
- Утилита *switch_root* **переносит** системные директории (/proc, /sys, /dev)
- Содержимое initramfs (tmpfs) рекурсивно **удаляется** для освобождения RAM
- Реальный корень становится **активным**
- Происходит вызов функции *exec* для запуска */sbin/init* на реальном диске


#### Рождение PID 1

- Процесс с идентификатором 1 является **прародителем** всех остальных **процессов**
- В Ubuntu 24.04 файл */sbin/init* является **симлинком** на */lib/systemd/systemd*
- Ядро переходит в **пассивный режим** обработки системных вызовов и прерываний
- Управление сервером полностью делегируется **системному менеджеру**
- Начинается фаза **асихронного** запуска пользовательских демонов


#### Философия и архитектура systemd

1) Отказ от скриптов в пользу декларативного **ini-подобного синтаксиса**
2) Агрессивная **параллелизация** задач при старте
3) Построение ациклического ориентированного графа (DAG) зависимостей
4) Использование **изоляции ядра** (cgroups и namespaces) из коробки
5) Единая **стандартизированная** плоскость управления сервером


#### Интеграция шины D-Bus

- **D-Bus** - система межпроцессорного взаимодействия (IPC)
- позволяет демонам асинхронно сообщать PID 1 о своей готовности (*Type=notify*)
- Устраняет необходимость в использовании костылей вроде sleep 5 в скриптах запуска
- Является основным API для взаимодействия с *systemd* из командной строки
- Поддерживает активацию сервисов по сигналу на шине
